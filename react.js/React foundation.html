The Core Mental Model & Mechanics

Thinking in Components

React is not a framework; it's a library for building User Interfaces (UI). We build UIs by combining small, independent, reusable pieces called Components.

The Rules

  • Components are just JavaScript functions.
  • They must return JSX (HTML-like syntax).
  • They must start with a Capital Letter. 
// 1. Define
function UserCard() {
  return (
    <div className="card">
      <h2>Alex Smith</h2>
      <p>Developer</p>
    </div>
  );
}

// 2. Use
export default function App() {
  return (
    <section>
      <UserCard />
      <UserCard />
    </section>
  );
}

Data Flow: Props vs. State

Props (Properties)

Arguments passed to a function.

  • Passed from Parent to Child.
  • Read-Only (Immutable).
  • Used to configure a component (color, text, id).

State

Variables inside a function.

  • Managed inside the component.
  • Can change over time (Mutable).
  • Triggers a Re-render when changed.

Interactive UI: useState

Standard variables (let x = 0) don't work in React because React doesn't know when they change. We use useState to tell React: "When this data changes, update the screen."

Syntax Anatomy

const [val, setVal] = useState(init);
  1. val: The current data.
  2. setVal: Function to update it.
  3. init: The starting value.
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>You clicked {count} times</p>
      
      <button onClick={() => setCount(count + 1)}>
        Click Me
      </button>
    </div>
  );
}

Architecture: The Virtual DOM

Updating the browser (Real DOM) is slow. React uses a clever trick called the Virtual DOM to be fast.

  • React keeps a copy of the UI in memory (JS Objects).
  • When state changes, it updates the virtual copy first.
  • It compares the copy to the real DOM (Diffing).
  • It only updates the specific text/pixels that changed.

"React is the agent that manages the DOM for you."

Side Effects: useEffect

Rendering UI should be "pure". But sometimes we need to talk to the outside world (API calls, Timers). We use Effects for this.

The Dependency Array

  • []: Run once on mount (Init).
  • [id]: Run when `id` changes.
  • Missing: Run every render (Bad!).
useEffect(() => {
  // 1. Logic (Fetching data)
  console.log("Component Mounted");

  const timer = setInterval(() => {}, 1000);

  // 2. Cleanup (Optional)
  return () => clearInterval(timer);
}, []); // <--- Dependencies
