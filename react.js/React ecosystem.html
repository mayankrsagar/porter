Routing, Fetching & Production Skills

Client-Side Routing (SPA)

In a Single Page App (SPA), we don't reload the browser to change pages. We just swap components using React Router.

Important Rule

Never use <a href="...">! It triggers a page refresh.

Use <Link to="..."> for instant transitions. 

import { Routes, Route, Link } from "react-router-dom";

function App() {
  return (
    <>
      <nav>
        <Link to="/">Home</Link>
      </nav>
      
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </>
  );
}

Data Fetching Patterns

Fetching data is asynchronous. Your component needs to handle the time before the data arrives.

  • Loading State: Show a spinner while waiting.
  • Success State: Show the data.
  • Error State: Show a message if it fails. 
if (isLoading) return <Spinner />;
if (error) return <ErrorMessage />;

return (
  <div>
    {data.map(item => (
      <div key={item.id}>{item.name}</div>
    ))}
  </div>
);

Forms: React Hook Form

Handling forms with standard React state (Controlled Components) is slow for large forms because every keystroke triggers a re-render.

The Solution

Use React Hook Form. It manages form state efficiently and includes validation (Required fields, Email format) out of the box.
